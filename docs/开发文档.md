# 开发文档 · 虚拟生物-兽医诊断游戏

## 2026-05-03：Web 游戏界面（P0）

| 文件 | 说明 |
|------|------|
| `data/cases.json` | 病例模板库（2个病例：肺炎/ARF） |

| 文件 | 改动 |

### 新增 API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/cases` | 获取病例列表 |
| GET | `/api/examinations` | 检查项目定义 |
| GET | `/api/treatments` | 治疗方案 |
| POST | `/api/new-game` | 开始游戏（注入疾病、创建 GameState） |
| POST | `/api/examine` | 开具检查（→ process_action → translate → 报告） |
| POST | `/api/diagnose` | 提交诊断+治疗（→ apply_treatment → 胜负判定） |
| POST | `/api/wait` | 等待观察（推进引擎 60 秒） |
| GET | `/api/game-state` | 获取当前游戏状态 |
| GET | `/api/hint` | 诊断提示（基于已收集线索匹配疾病） |

### 前端技术选型

**Vue 3（CDN）**，理由：
1. 游戏是状态驱动 UI，Vue 响应式数据绑定天然适合
2. CDN 引入，零构建工具，和现有 Flask 部署方式一致
3. 组件化拆分自然（VitalCard、ExamGrid、ReportList 等）
4. `{% raw %}` 包裹解决 Vue `{{ }}` 与 Jinja2 冲突

### 架构关系

```
gui_app.py (Flask)
  ├── /api/*  →  JSON  ←  game.html (Vue 3)
  │                              ├── VitalCard 组件
  │                              ├── ExamGrid（检查开具）
  │                              ├── ReportList（报告折叠展示）
  │                              └── DiagnosisPanel（诊断+治疗）
  └── action_system.py
        ├── process_action()  ←  推进引擎 60 秒
        ├── determine_phase() ←  综合评分（MAP/SpO2/HR/pH/DO2/乳酸/尿量）
        └── check_death()     ←  濒死倒计时
```
