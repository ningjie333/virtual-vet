## BioFlow Labs v2 Jobs List (Execution Plan)

---

### Phase 0 — Repo + Modules (45–90 min)
----------------------------------------------------------------------------
**Done when:** imports are clean and boundaries enforced.

- Create v2 folder structure (core/, engine_app/, db/, tools/)
- Add versioning: ENGINE_VERSION
- Decide canonical units list (document it once)
- Add basic logging infrastructure (structured logs)

---

### Phase 1 — Template Schema v2.0 + Validation (2–4 hrs)
----------------------------------------------------------------------------
**Done when:** invalid templates are rejected with precise errors.

- Write JSON Schema for v2.0 template
- Define “runnable” rules:
  - required fields present
  - numeric bounds (no negative volumes, etc.)
  - conservation constraints (initial volumes sum to total)
- Validation output:
  - is_valid
  - errors[]
  - warnings[]
  - template_hash
- Add fixtures:
  - valid baseline adult
  - young athlete (factory result)
  - smoker-ish (factory result)
  - intentionally broken templates for tests

**Tests**
- Schema pass/fail tests
- Bounds tests
- Hash reproducibility tests

---

### Phase 2 — Database v1 (Templates + Runs) (2–5 hrs)
----------------------------------------------------------------------------
**Done when:** engine can store, query, and delete invalid templates safely.

**Tables (minimum):**

**templates**
- id, name, created_at
- json (raw)
- template_version
- is_valid (bool)
- validation_errors (json/text)
- template_hash (unique)

**runs**
- id, started_at, ended_at
- template_hash, template_snapshot_json
- engine_version
- run_config_json (dt, duration, sample_rate)
- summary_json (computed metrics)

**run_samples** (optional but recommended early)
- run_id, t_ms
- global_state_json (or columnar later)

**run_events**
- run_id, t_ms, level, code, message

**Jobs**
- DB init function (creates tables if missing)
- CRUD:
  - insert template (store even if invalid)
  - list runnable templates (valid only)
  - delete invalid templates
  - fetch template by id/hash
- Run persistence:
  - create run row at start
  - append samples/events
  - finalize run with summary

**Tests**
- DB creates cleanly on new machine
- Insert invalid template: stored, flagged, not runnable
- Delete invalid templates: only invalid removed
- Run write path works

---

### Phase 3 — Engine Runner: Load → Validate → Run → Log (2–4 hrs)
----------------------------------------------------------------------------
**Done when:** you can run from DB ID OR JSON file OR direct object and get a run record.

**Entry points:**
- run default template
- run template by DB id
- run template from JSON file path

**Runner pipeline:**
- load template
- validate (hard fail if not runnable)
- build sim state from initial_state or defaults
- execute for duration
- write samples/events + summary to DB

**Tests**
- Same template twice → identical summary hash
- Attempt run invalid template → handled error, no partial run

---

### Phase 4 — Physiology v2: Multi-bed Circulation (6–12 hrs)
----------------------------------------------------------------------------
**Done when:** multi-bed competition is stable, conserved, and visibly different across beds.

**Jobs**
- Implement bed abstraction:
  - parameters: R, C, V0, optional pooling bias
  - state: V, P, Q
- Implement parallel flow solver (keep it simple first):
  - Q_bed = (P_art - P_ven) / R_bed
  - (then modify with compliance dynamics)
- Compliance dynamics:
  - P = (V - V0) / C (per compartment)
- Venous pooling modeling:
  - simplest v2: periphery bed has larger V0 / compliance and captures volume under standing toggle
- Perfusion index:
  - baseline flow computed from template baseline
  - index = clamp(100 * Q / Q_baseline)

**Tests**
- Conservation tests across all beds
- Extremes don’t explode (high R, low C)
- Bed competition:
  - raising one bed’s R reduces its flow and redistributes to others

---

### Phase 5 — Posture + Stress knobs (optional but high value) (2–6 hrs)
----------------------------------------------------------------------------
**Done when:** standing vs supine changes pooling and perfusion in a familiar way.

- Posture toggle in template (resolved param)
- Standing increases peripheral pooling tendency / venous capacitance bias
- Mild hemorrhage control:
  - reduce total_blood_volume_ml (template scenario variant)
- Global vasoconstriction factor:
  - multiply systemic bed resistances

**Tests**
- Standing increases pooled volume and reduces venous return (emergent)
- Hemorrhage lowers pressures and perfusion indices

## v2 “Definition of Done” (non-negotiable)
----------------------------------------------------------------------------
- DB auto-creates on first launch.
- Templates can be stored even if invalid.
- Engine refuses invalid templates with a clean error.
- Engine writes a run log with:
  - template snapshot + hash
  - time series samples (at least global)
  - summary metrics
- Multi-bed competition works and stays stable.
- Deterministic replay: same inputs → same outputs.
